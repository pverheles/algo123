Brute Force

Try all possibilities and pick the best or first that satisfies the condition.

Use when
  • Search space is small
  • As a baseline to verify other methods
Examples
  • Linear search
  • Checking all permutations

Divide & Conquer

Split the problem into subproblems, solve them recursively, and combine results.

Use when
  • Problem splits naturally
  • Combine step is efficient
Examples
  • Merge sort
  • Binary search
  • Closest pair of points
Pitfalls
  • Expensive combine step
  • Unbalanced splits

Greedy

Build the solution step by step by making the locally optimal choice at each step.

Use when
  • Greedy choice property holds
  • Optimal substructure
Examples
  • Interval scheduling
  • Huffman coding
  • Prim/Kruskal MST
Pitfalls
  • Fails without proof of correctness
  • Counterexamples common
