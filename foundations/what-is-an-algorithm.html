What is an Algorithm?

A precise, finite sequence of steps that transforms input into the desired output. In practice, a good algorithm is correct, efficient, definite, and terminating.

Definition & Intuition

An algorithm is a well‑defined procedure that takes some value(s) as input and produces some value(s) as output. It must be unambiguous and always finish. Think of it like a recipe: ingredients (inputs), steps (procedure), and dish (output).

Examples
  • Long division (math)
  • Euclid’s GCD
  • Merge sort (CS)
  • Tea recipe (everyday)
Deterministic vs Randomized
  • Deterministic: same input → same steps
  • Randomized: uses randomness (e.g., randomized quicksort)

Key Properties

  • Correctness — solves the intended problem for all valid inputs.
  • Efficiency — reasonable time/space usage (scales with input size).
  • Finiteness — terminates after a finite number of steps.
  • Definiteness — each step is clear and unambiguous.
  • Input/Output — clearly specified inputs and at least one output.

Algorithm vs Program vs Heuristic

Algorithm

Abstract, language‑agnostic set of steps. Can be expressed as pseudocode, math, or a diagram.

Program

A concrete implementation of one (or more) algorithms in a programming language, runnable by a computer.

Heuristic

A practical rule‑of‑thumb that often works but may not guarantee optimality or correctness in all cases.

Pseudocode Conventions

Pseudocode describes the algorithm precisely without being tied to a specific language or compiler.

// Linear Search
LINEAR-SEARCH(A, x)
  for i ← 0 to length(A) − 1 do
    if A[i] = x then
      return i
  return NOT-FOUND

// Binary Search (requires sorted A)
BINARY-SEARCH(A, x)
  low ← 0; high ← length(A) − 1
  while low ≤ high do
    mid ← ⌊(low + high) / 2⌋
    if A[mid] = x then return mid
    else if A[mid] < x then low ← mid + 1
    else high ← mid − 1
  return NOT-FOUND

Correctness (sketch)

To argue that an algorithm always gives the right answer and terminates:

  • Preconditions: what must be true before it runs (e.g., array is sorted for binary search).
  • Postconditions: what must be true after it runs (e.g., returned index points to the key or NOT‑FOUND).
  • Loop invariants: facts that remain true on every iteration (used with induction: initialization, maintenance, termination).
  • Termination: show a measure ("rank") strictly decreases and is bounded below.

Efficiency (time & space)

We analyze resource growth with input size n using asymptotic notation:

  • O(1) constant, O(log n) logarithmic
  • O(n) linear, O(n log n) near‑linear
  • O(n²) quadratic, …

Binary search beats linear search on large, sorted inputs: O(log n) vs O(n).
