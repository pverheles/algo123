What is an Algorithm?

A precise, finite sequence of steps that transforms input into the desired output. In practice, a good algorithm is correct, efficient, definite, and terminating.

Definition & Intuition

An algorithm is a well‚Äëdefined procedure that takes some value(s) as input and produces some value(s) as output. It must be unambiguous and always finish. Think of it like a recipe: ingredients (inputs), steps (procedure), and dish (output).

Examples
  • Long division (math)
  • Euclid‚Äôs GCD
  • Merge sort (CS)
  • Tea recipe (everyday)
Deterministic vs Randomized
  • Deterministic: same input ‚Üí same steps
  • Randomized: uses randomness (e.g., randomized quicksort)

Key Properties

  • Correctness ‚Äî solves the intended problem for all valid inputs.
  • Efficiency ‚Äî reasonable time/space usage (scales with input size).
  • Finiteness ‚Äî terminates after a finite number of steps.
  • Definiteness ‚Äî each step is clear and unambiguous.
  • Input/Output ‚Äî clearly specified inputs and at least one output.

Correctness (sketch)

To argue that an algorithm always gives the right answer and terminates:

  • Preconditions: what must be true before it runs (e.g., array is sorted for binary search).
  • Postconditions: what must be true after it runs (e.g., returned index points to the key or NOT‚ÄëFOUND).
  • Loop invariants: facts that remain true on every iteration (used with induction: initialization, maintenance, termination).
  • Termination: show a measure ("rank") strictly decreases and is bounded below.
