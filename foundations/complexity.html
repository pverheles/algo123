Why Complexity?

Complexity analysis compares algorithms independently of hardware by focusing on how resource usage grows with input size n. It guides design choices and scalability expectations.

Time

How many basic steps (comparisons, assignments) as a function of n.

Space

How much extra memory (beyond input) does the algorithm require?

Best / Average / Worst Cases

  • Worst‚Äëcase: maximum cost for inputs of size n (often used for guarantees).
  • Average‚Äëcase: expected cost over all inputs of size n (needs input distribution model).
