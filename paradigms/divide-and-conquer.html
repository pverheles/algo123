Divide & Conquer

Split ‚Üí solve subproblems ‚Üí combine. A powerful pattern behind merge sort, fast exponentiation, FFT, closest pair of points, and more.

Definition

Divide & Conquer solves a problem by splitting it into smaller subproblems of the same kind, solving each recursively, and combining their solutions.

When to use

Natural splits

Problem decomposes into similar subproblems.

Efficient combine

Merging/combining takes near‚Äëlinear time.

Balanced sizes

Subproblems are of roughly equal size.

Parallelizable

Independent subproblems can be parallelized.

Design Template

DIVIDE-AND-CONQUER(Problem P)
  if small(P) then
    return solveDirectly(P)
  split P into P1, P2, ..., Pa
  S1, S2, ..., Sa ‚Üê recursively solve each Pi
  return combine(S1, S2, ..., Sa)

Recurrence & Master Theorem

Running time often satisfies \(T(n) = a\,T(n/b) + f(n)\), where \(a\) is the number of subproblems, each of size \(n/b\), and \(f(n)\) is split+combine cost.

Case 1

If \(f(n) = O(n^{\log_b a - \varepsilon})\), then \(T(n) = \Theta(n^{\log_b a})\).

Case 2

If \(f(n) = \Theta(n^{\log_b a}\,(\log n)^k)\), then \(T(n) = \Theta(n^{\log_b a}\,(\log n)^{k+1})\).

Case 3

If \(f(n) = \Omega(n^{\log_b a + \varepsilon})\) (plus regularity), then \(T(n) = \Theta(f(n))\).

Examples

Merge sort

\(T(n)=2T(n/2)+n=\Theta(n\log n)\).

Binary search

\(T(n)=T(n/2)+1=\Theta(\log n)\).

Closest pair

Split plane by median; combine with strip check ‚Üí \(\Theta(n\log n)\).

Karatsuba

\(T(n)=3T(n/2)+n=\Theta(n^{\log_2 3})\approx n^{1.585}\).

Pitfalls

  • Highly unbalanced splits (degrades to \(T(n)=T(n-1)+f(n)\)).
  • Expensive combine step can dominate.
  • Base cases not tuned ‚Üí overhead.
